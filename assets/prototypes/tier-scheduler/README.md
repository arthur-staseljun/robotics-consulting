# Планировщик шиномонтажника

Приложение помогает быстро оформить запись клиента во время звонка: увидеть входящий номер, открыть экран действий, выбрать слот и сохранить бронь.

## Кому это нужно

- Владелец или администратор шиномонтажа.
- Сотрудник, который принимает входящие звонки и сразу записывает клиентов.

## Что такое интеграция звонков

**Интеграция звонков** — это режим, при котором приложение получает доступ к событиям звонка (входящий/активный/завершен) и может показать экран действий по звонку.

### Зачем включать интеграцию

1. Чтобы не переключаться между телефоном и календарем вручную.
2. Чтобы во время звонка сразу видеть номер и доступные слоты.
3. Чтобы быстрее сохранять запись, снижая риск ошибок и пропущенных клиентов.

### Когда нужно включать или выключать интеграцию со звонками

- **После первой установки** приложения.
- **Если экран звонка перестал открываться автоматически**.
- **Если необходимо обычное поведение звнков**.

## Инструкция после установки (очень подробно)

### Шаг 1. Первый запуск приложения

1. Откройте `Планировщик шиномонтажника`.
2. Разрешите запрошенные права (контакты, уведомления — если запрашиваются).
3. На экране интеграции звонков выберите приложение как телефон по умолчанию (default dialer), если появится запрос.

### Шаг 2. Проверьте роль приложения для звонков

1. Откройте раздел интеграции в приложении.
2. Убедитесь, что приложение назначено как обработчик звонков.
3. Если нет — нажмите кнопку включения интеграции и подтвердите выбор.

### Шаг 3. Включите системные разрешения (особенно важно на Xiaomi/MIUI/HyperOS)

1. **Автозапуск**
   - `Настройки` -> `Приложения` -> `Разрешения` -> `Автозапуск`
   - включите для `Планировщик шиномонтажника`.

2. **Показ поверх других приложений / всплывающие в фоне**
   - `Настройки` -> `Приложения` -> `Все приложения` -> `Планировщик шиномонтажника` -> `Другие разрешения`
   - включите:
     - `Показывать поверх других приложений`
     - `Отображать всплывающие окна в фоне`.

3. **Ограничения батареи**
   - `Настройки` -> `Батарея` -> `Экономия батареи приложений`
   - для приложения выберите `Без ограничений` (`No restrictions`).

4. **Уведомления звонка**
   - `Настройки` -> `Уведомления` -> `Уведомления приложений` -> `Планировщик шиномонтажника`
   - в канале `incoming_call_channel` включите:
     - всплывающие уведомления (heads-up),
     - показ на экране блокировки,
     - высокий приоритет,
     - полноэкранный показ (если доступно).

### Шаг 4. Тестовый звонок

1. Закройте приложение или сверните в фон.
2. Сделайте тестовый входящий звонок.
3. Ожидаемый результат:
   - открывается экран входящего звонка в приложении,
   - можно принять/отклонить,
   - можно перейти к созданию записи.

## Как пользоваться каждый день

### При входящем звонке

1. Откройте экран звонка в приложении (автоматически или через уведомление).
2. Посмотрите номер и данные клиента (если есть в контактах).
3. Нажмите действие для записи и выберите подходящий слот.
4. Сохраните запись.

### Управление записями

1. В главном экране выберите дату.
2. Нажмите на свободный слот, чтобы добавить запись.
3. Нажмите на занятой слот, чтобы изменить апись, на крестик, чтобы отменить.
4. При необходимости блокируйте время (выходной, перерыв, занятость бокса).

### Работа с клиентами (новая модель)

- Все данные клиентов (имя, фамилия, телефон, номер машины, хранение шин) хранятся **только во внутренней базе приложения**.
- Системные контакты не изменяются автоматически.
- При первом звонке имя/фамилия могут быть взяты из контакта, но далее редактируются и сохраняются только в базе приложения.
- Информация о хранении шин (дата начала/окончания) также хранится в базе приложения.

### Экспорт/импорт записей

- Для экспорта всех записей откройте `⚙` на главном экране, прокрутите вниз до блока `Экспорт/Импорт данных` и нажмите `Экспорт`.
- Файл `bookings_export.csv` сохраняется во внутреннем хранилище приложения (`/data/data/com.robotics.consulting.app/files/`).
- Формат CSV: UTF-8, экранирование по стандарту (кавычки удваиваются, поля с запятыми/переводами строки берутся в кавычки).
- Для копирования экспортного файла используйте ADB:

```zsh
adb shell run-as com.robotics.consulting.app cp files/bookings_export.csv /sdcard/bookings_export.csv
adb pull /sdcard/bookings_export.csv ./bookings_export.csv
```

- Для импорта подготовьте файл `bookings_import.csv` в таком же CSV-формате и загрузите его во внутренние файлы приложения:

```zsh
adb push ./bookings_import.csv /sdcard/bookings_import.csv
adb shell run-as com.robotics.consulting.app cp /sdcard/bookings_import.csv files/bookings_import.csv
```

- После этого в приложении откройте блок `Экспорт/Импорт данных` и нажмите `Импорт`.

## Почему это выгодно бизнесу (для тех, кто выбирает и покупает систему)

`Планировщик шиномонтажника` — это не только календарь записи, а операционная система точки: звонок -> запись -> услуга -> повторный визит.

### Что получает владелец сервиса

- Прозрачность загрузки: видно, какие дни и часы реально приносят выручку.
- Контроль услуг: сколько было `Замена шин`, сколько `Замена масла`, где есть потенциал роста среднего чека.
- Управление командой: проще планировать смены и загрузку постов по историческим данным.
- Повторные продажи: база записей помогает работать не «по памяти», а по фактам.

### Чем полезен экспорт записей

Экспорт `bookings_export.csv` позволяет делать простой, но эффективный бизнес-анализ в Excel/Google Sheets/BI:

- Анализ пиков по дням и часам.
- Планирование персонала и графиков без перегруза.
- Сегментация клиентов по типу услуги и частоте визитов.
- Оценка конверсии из входящего звонка в фактическую запись.

### Потенциал роста выручки (upsell/cross-sell)

Если добавить в модель записи поле `глубина протектора`, можно построить проактивные продажи:

- При звонке сразу видеть, кому скоро нужна замена шин.
- Предлагать комплект + установку по спеццене во время записи.
- Делать «умные» сезонные предложения (осень/весна) по клиентам из базы.

Это превращает приложение из «журнала записи» в инструмент роста выручки и управляемости сервиса.

## Если что-то не работает

### Ситуация: "экран не открылся автоматически"

1. Проверьте шаг 3 (автозапуск, фоновые окна, батарея, уведомления).
2. Убедитесь, что интеграция звонков включена в приложении.
3. Сделайте повторный тестовый звонок.

### Ситуация: "приходит только уведомление"

Это допустимо на некоторых прошивках: система может блокировать прямой запуск экрана из фона.
В таком случае нажмите уведомление звонка — приложение откроется и позволит оформить запись.

- После установки на новом телефоне обязательно прогоните 1-2 тестовых звонка.

#### Практический playbook на выезде (по ошибкам)

**1) `INSTALL_FAILED_VERSION_DOWNGRADE`**

Причина: на устройстве уже стоит APK с `versionCode` выше текущей сборки.

Что делать:
1. Удали установленное приложение.
2. Поставь debug заново.

```zsh
adb uninstall com.robotics.consulting.app
cd "/Users/daiver/IdeaProjects/wheel-time-tracker"
./gradlew :app:installDebug
```

---

**2) `INSTALL_FAILED_UPDATE_INCOMPATIBLE` / `...SIGNATURE`**

Причина: старая версия подписана другим ключом (например release/другая debug-подпись).

Что делать:
1. Удали старую версию с устройства.
2. Поставь текущую сборку заново.

```zsh
adb uninstall com.robotics.consulting.app
cd "/Users/daiver/IdeaProjects/wheel-time-tracker"
./gradlew :app:installDebug
```

---

**3) `INSTALL_FAILED_INSUFFICIENT_STORAGE`**

Причина: не хватает места в памяти телефона.

Что делать:
1. Проверить свободное место.
2. Очистить место (медиа/кеш/ненужные приложения).
3. Повторить установку.

```zsh
adb shell df -h /data
cd "/Users/daiver/IdeaProjects/wheel-time-tracker"
./gradlew :app:installDebug
```

Если всё ещё не хватает места, временно удалить старую версию приложения:

```zsh
adb uninstall com.robotics.consulting.app
./gradlew :app:installDebug
```

---

**4) `unauthorized` в `adb devices`**

Причина: телефон не подтвердил RSA-ключ отладки USB.

Что делать:
1. Разблокируй экран телефона и подтверди диалог `Разрешить отладку по USB`.
2. Если диалог не появляется: на телефоне в Developer options нажми `Revoke USB debugging authorizations`.
3. Перезапусти ADB на ноутбуке и проверь статус заново.

```zsh
adb kill-server
adb start-server
adb devices
```

Если всё ещё `unauthorized`: смени кабель/USB-порт и убедись, что режим USB — `Передача файлов (MTP)`.

### Как этим пользоваться ###
- При доработке звонков смотри сначала CallStateReducer и coordinator'ы из presentation — там "правила", а не Android side-effects.
- MainActivity теперь выступает как orchestrator: принимает решения из coordinator'ов и применяет эффекты (launch, toast, permission request).
- Для контактов:
    - бизнес-логика в `ContactSyncService`
    - Android-специфика в `AndroidContactStore`
    - в тестах используем fake `ContactStore`.

## Удалено: Бэкапы и запись в системные контакты

- Приложение больше не изменяет заметки или другие поля системных контактов.
- Все данные клиента теперь независимы от телефонной книги.
- Старые инструкции по бэкапу контактов и восстановлению больше не актуальны.
